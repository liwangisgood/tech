    基本数据类型的默认值是0或false，而包装数据类型的默认值是null。

    包装数据类型可以使用一些额外的方法，如equals()和hashCode()等。

12.关于索引的设计，下列哪些说法符合《阿里巴巴Java开发手册》：【D】多选ACD

  A .对varchar类型的字段建立索引，必须指定索引长度。:11
  (MySql字段索引长度建立语句
  在MySQL中，可以使用以下语句来为VARCHAR类型的字段建立索引，并指定索引长度：
  CREATE INDEX index_name ON table_name (column_name(length));
  其中，indexname是索引的名称，tablename是表的名称，column_name是要建立索引的字段名称，length是索引的长度。例如，要为表user的name字段建立长度为50的索引，可以使用以下语句：
  CREATE INDEX idx_user_name ON user (name(50));
  需要注意的是，索引长度不能超过字段的最大长度，否则会导致索引失效或者查询效率降低。因此，在选择索引长度时，需要根据具体的业务需求和数据量来决定。
需要注意的是，字段长度只是用于限制字段的最大长度，实际存储的数据可以比字段长度小，但不能超过字段长度。如果存储的数据超过了字段长度，MySQL会自动截取数据或者报错。因此，在定义字段长度时，需要根据实际业务需求和数据量来选择合适的长度。)

  B .对varchar类型的字段建立索引，不需要指定索引长度，这样索引区分度最好。:11

  C .业务上具有唯一特性的字段（含组合字段），必须指定唯一索引。:11
  (MySql什么是唯一特性的字段（含组合字段)?
  在MySQL中，唯一特性的字段是指具有唯一性约束的字段，即该字段的值在表中必须是唯一的，不能重复。唯一特性的字段可以是单个字段，也可以是多个字段的组合。例如，在用户表中，用户名和邮箱地址都是唯一的，不能重复，因此可以将它们作为组合字段来创建唯一索引。在MySQL中，可以使用UNIQUE关键字来创建唯一索引，以保证唯一特性的字段的唯一性约束。唯一索引可以包含多个列，也可以是单列索引。在创建唯一索引时，需要确保索引列的值是唯一的，否则会导致插入数据失败或者更新数据时出现冲突。唯一特性的字段在数据库设计中非常重要，它可以保证数据的完整性和一致性，避免数据的重复和冲突。
  唯一约束或字段要求这些组合值必须是唯一)
  (唯一索引和唯一约束都可以用来保证字段或字段组合的唯一性，避免数据的重复和冲突。但是，它们的实现方式略有不同：
  唯一索引是一种物理结构，它会在数据库中创建一个索引来保证字段或字段组合的唯一性。当我们在表中创建唯一索引时，MySQL会自动为该索引创建一个B+树结构，用于快速查找和验证唯一性。唯一索引可以提高查询效率，但会占用一定的存储空间。
  唯一约束是一种逻辑结构，它会在数据库中创建一个约束来保证字段或字段组合的唯一性。当我们在表中创建唯一约束时，MySQL会在插入或更新数据时自动验证唯一性，如果违反唯一性约束，则会返回一个错误。唯一约束不会占用额外的存储空间，但可能会降低查询效率。

在设计数据库时，应该根据具体的业务需求和数据特性来选择使用唯一索引还是唯一约束。如果需要提高查询效率，可以使用唯一索引；如果只需要保证数据的完整性和一致性，可以使用唯一约束。)

  D .建复合索引时，一般选择区分度高的字段放在最左列。

14.关于索引的使用，下列哪些说法是正确的：【】多选

  A .查询语句 WHERE a+1 = 5 可以利用a索引。

  B .查询语句WHERE date_format(gmt_create, '%Y-%m-%d') = '2016-11-11'无法利用gmt_create索引。

  C .当 c 列类型为 char 时，查询语句 WHERE c = 5 无法利用c索引。

  D .索引字段使用时不能进行函数运算。
  (A. 查询语句 WHERE a+1 = 5 可以利用a索引。这个说法是错误的。当查询语句中对索引字段进行运算时，MySQL无法直接使用索引，因为索引是按照字段的值进行排序的，而不是按照运算结果进行排序的。在这种情况下，MySQL会放弃使用索引，而是对所有数据进行全表扫描，这会导致查询效率变慢。因此，如果要使用索引，应该尽量避免对索引字段进行运算。
B. 查询语句WHERE dateformat(gmtcreate, '%Y-%m-%d') = '2016-11-11'无法利用gmt_create索引。
这个说法是正确的。当查询语句中对索引字段进行函数运算时，MySQL无法直接使用索引，因为函数运算会改变字段的值，使得索引失效。在这种情况下，MySQL会放弃使用索引，而是对所有数据进行全表扫描，这会导致查询效率变慢。因此，如果要使用索引，应该尽量避免对索引字段进行函数运算。
C. 当 c 列类型为 char 时，查询语句 WHERE c = 5 无法利用c索引。

这个说法是正确的。当查询语句中对字符类型的字段进行数值比较时，MySQL会将字符类型的字段转换为数值类型进行比较，这会导致索引失效。在这种情况下，MySQL会放弃使用索引，而是对所有数据进行全表扫描，这会导致查询效率变慢。因此，如果要使用索引，应该尽量避免对字符类型的字段进行数值比较。
D. 索引字段使用时不能进行函数运算。

这个说法是正确的。当查询语句中对索引字段进行函数运算时，MySQL无法直接使用索引，因为函数运算会改变字段的值，使得索引失效。在这种情况下，MySQL会放弃使用索引，而是对所有数据进行全表扫描，这会导致查询效率变慢。因此，如果要使用索引，应该尽量避免对索引字段进行函数运算。)

8、关于参数校验,下列哪些说法符合《阿里巴巴Java开发手册》【】

B.需要极高稳定性和可用性的方法需要对参数进行校验

文档：【推荐】接口入参保护，这种场景常见的是用于做批量操作的接口。

【参考】方法中需要进行参数校验的场景：

1） 调用频次低的方法。

2） 执行时间开销很大的方法，参数校验时间几乎可以忽略不计，但如果因为参数错误导致中间执行回退，或者错误，那得不偿失。

3） 需要极高稳定性和可用性的方法。

4） 对外提供的开放接口，不管是 RPC/API/HTTP 接口。

【参考】方法中不需要参数校验的场景：

1） 极有可能被循环调用的方法，不建议对参数进行校验。但在方法说明里必须注明外部参数检查。

2） 底层的方法调用频度都比较高，一般不校验。毕竟是像纯净水过滤的最后一道，参数错误不太可能到底层才会暴露问题。一般 DAO 层与 Service 层都在同一个应用中，部署在同一台服务器中，所以 DAO 的参数校验，可以省略。

3） 被声明成 private 只会被自己代码所调用的方法，如果能够确定调用方法的代码传入参数已经做过检查或者肯定不会有问题，此时可以不校验参数。
