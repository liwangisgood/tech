  增删改 git remote add,git remote rm,git remote rename,git remote set-url

历史版本 
  git底层玩的是指针：切换版本、切换分支head指向(head只想当前分支)
  git reset切换版本底层移动head指针，本地文件回退历史文件文件夹
  git reflog,git log,git reset --hard
  merge冲突，<<<<HEAD    当前分支数据如：master     ============   合并的分支hot-fix数据   >>>>>> hot-fix
  1、在冲突文件a.txt 删除所有其他符号<<<<   ====    >>>>，或者不想要的数据
  2、git add a.txt  , git commit -m "conflict merge commit" 不要a.txt文件名（cannot do a partial commit during a merge） , merging状态变为正常状态
